ROM_Handler Implementation Notes

The current implementation of the ROM_Session is quite crude and does not account for checking sequence numbers.  There is currently a preprocessor definition that needs to be added to get some checking on session-level sequence numbers; that definition is SUPPORTS_CSV_SEQ_NUMS.

An additional question is to what extent these messages will be persisted.  We can either fulfill resends simply from what's in-memory, which is fine if the process is not restarted or doesn't crash intraday.  Or we could implement some sort of file-backed store to store the messages for later consumption.  This has the advantage of being able to offload the storage of old and dead transactions to disk instead of storing it all in memory in the off-chance the client reconnects and wants a resend.

Configuration

The ROM_Handler will need a configuration source, ideally one that is somewhat common with other components in the process and therefore common across many processes in the system, if this is to be reused.  The ACE framework has a configuration system, will need to explore that.  For now, an INI file will probably do.
